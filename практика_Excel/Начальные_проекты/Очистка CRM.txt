ОПИСАНИЕ КЕЙСА: СИСТЕМА КОНТРОЛЯ КАЧЕСТВА ДАННЫХ CRM
Общая информация
Название проекта: Разработка системы контроля качества и стандартизации клиентской базы CRM

Цель проекта: Создать прозрачную, автоматизированную систему для управления качеством данных клиентов с сохранением полной истории изменений

Период выполнения: Декабрь 2024

Инструменты: Microsoft Excel 2021 (версия 2510)

Проблема бизнеса

Компания столкнулась с низким качеством данных в клиентской базе CRM:
Неполные записи клиентов
Нестандартные форматы данных
Отсутствие единых правил заполнения
Невозможность точного анализа из-за "мусорных" данных
Риск принятия неверных решений на основе некорректной информации

Поставленные задачи

Анализ текущего состояния данных без их удаления или искажения
Стандартизация форматов ключевых полей (даты, email, географические данные)
Создание системы классификации записей по степени готовности к использованию
Разработка механизма автоматической проверки новых и существующих записей
Обеспечение прозрачности — любой пользователь должен понимать, почему запись имеет тот или иной статус

Выполненные работы
1. Подготовительный этап
Проведен аудит 20 записей клиентской базы

Выявлены 7 типов проблем:

Пустые обязательные поля
Нестандартные форматы дат
Сокращенные названия городов
Неполные email-адреса
Дублирующийся записи
Различные форматы ФИО
Отсутствие структуры в данных

2. Стандартизация данных
Унификация форматов:

Все даты приведены к единому формату ГГГГ-ММ-ДД
Названия городов стандартизированы: "Мск" → "Москва", "СПб" → "Санкт-Петербург"
Email-адреса дополнены доменами: "@mail" → "@mail.ru", "@gmail" → "@gmail.com"
Структурирование информации:
Поле "ФИО" разделено на три отдельных столбца: "Фамилия", "Имя", "Отчество"
Сохранены все исходные данные в исправленном виде
Устранены явные дубликаты с сохранением первой валидной записи

3. Разработка системы контроля качества
Создана автоматическая система проверки с двумя ключевыми столбцами:

а) Столбец "Статус записи" — трехуровневая классификация:

text
✅ Готово        — запись полностью заполнена и готова к использованию
❌ Есть проблемы — имеются пропуски в обязательных полях
⚠️ Пустая строка — запись содержит минимальную информацию
б) Столбец "Проблемы" — детализация для каждой записи:

Автоматическое формирование списка конкретных проблем

Формат: "● [Тип проблемы] | " (например: "● Нет имени | ● Нет email | ")

Проверяются 7 ключевых полей: Фамилия, Имя, Отчество, Email, Город, Дата регистрации, Сумма покупки

4. Визуализация и удобство использования
Цветовое кодирование статусов для мгновенной визуальной оценки

Сохранение всех исходных записей — принцип "не удалять, а маркировать"

Возможность фильтрации по типу статуса для разных рабочих задач

Прозрачность — любой пользователь видит причины проблем

Выводы:

Данный проект демонстрирует системный подход к решению проблемы качества данных. Вместо разовой "очистки" была создана устойчивая система, которая:

Работает постоянно — автоматически проверяет новые и существующие данные
Прозрачна — каждый пользователь понимает причины проблем
Безопасна — не удаляет данные, а маркирует их
Практична — решает реальные бизнес-задачи
Масштабируема — может быть применена к любому объему данных

Главный принцип проекта: "Качество данных — это не разовое мероприятие, а непрерывный процесс, требующий системы, а не героических усилий отдельных сотрудников."
